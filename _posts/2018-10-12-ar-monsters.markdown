As a developer, finding new ways to combine technical skills and creativity is always a fun challenge. Recently, I had the opportunity to create a small proof of concept (POC) that involved using Pokemon cards and the Swift programming language to add 3D Pokemon models. The project was very basic and only allowed for the 3D models to be added over the top of the cards using SKAnchor points, but it was still an exciting first step.

The project started with the idea to create a unique and engaging user experience that would combine the nostalgia of Pokemon cards with the immersive world of 3D models. The first step was to create a set of placeholder images using the Pokemon cards. These images would represent each Pokemon and be used as a reference for the 3D models.

Next, I used Swift and the SKAnchor point library to create the 3D models. SKAnchor points are special reference points that allow developers to anchor 3D models to the real world. By using SKANchor points and the Pokemon card placeholder images, I was able to create a basic POC that allowed users to view 3D models of the Pokemon by holding their phone over the corresponding card.

The POC was very basic, but it was still an exciting accomplishment. Users could now view their favorite Pokemon in 3D by simply holding their phone over the corresponding card. But I didn't want to stop there. In the future i planned to take the project further by adding health bars and the ability to show the Pokemon's attacks as pressable buttons.

In the end, the POC was a simple yet effective concept that combined the nostalgia of Pokemon cards with the immersive world of 3D models. Although it was a basic project, it opened the door to new possibilities and gave me a taste of what's possible with Swift and ARKit. The project also showed the potential for creating engaging user experiences that combine real-world objects with interactive 3D models. As technology continues to advance, the possibilities for creating new and innovative apps are endless.
